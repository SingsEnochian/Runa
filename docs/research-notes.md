## 1. Research posture

We are treating telepathy, remote viewing, and related work as:
- interesting
- historically significant
- experimentally controversial

The lab posture is:
- curious

## 2. Telepathy

### Best-known laboratory paradigm
The classic modern telepathy paradigm is the ganzfeld procedure:
- receiver in sensory homogenization
- sender or target isolated
- random target selection
- blind judging or forced-choice selection

Why it matters for this repo:
- it is compatible with tone-based state induction
- it rewards careful logging
- it is one of the cleaner task families for pairing with focus-state shells

### Evidence shape
The evidence base is mixed but not empty.
There are meta-analytic summaries reporting small above-chance effects, but debate remains about:
- publication bias
- study selection
- quality variance
- replication strength

Operational takeaway:
If we build a telepathy mode, it should be designed like an experiment dashboard, not a proof altar.

## 3. Remote viewing

### Why it fits the Sound Lab
Remote viewing tasks pair well with:
- calm but alert states
- low verbal interference
- image-first reporting
- later judging against hidden targets

That makes it a strong candidate for:
- F12 shell testing
- image-first sessions
- response-confidence tracking
- outbound vs target-pool comparison experiments

### Evidence shape
Remote viewing has both:
- supportive meta-analytic literature reporting above-chance performance
- strong criticism regarding operational usefulness, replication, and interpretation

Operational takeaway:
The safest design target is not “spycraft.”
It is:
- structured target presentation
- sealed randomization
- image-first capture
- blind judging
- confidence and latency tracking

## 4. Focus-level note: Focus 33

Current official Monroe public materials clearly document public use of:
- F10
- F12
- F15
- F18
- F21
- F27
- F34/35
- F49

At present, Focus 33 does not appear to be a clearly documented current official Monroe public level in the same way those do.

Working hypothesis:
- the “Focus 33” audio encountered on YouTube was likely unofficial, fan-labeled, legacy-labeled, or platform-specific rather than a currently documented Monroe public level
- the closest official public neighbors now visible are F27 and F34/35

Practical implication for the lab:
If a user strongly resonated with a “Focus 33” track, the useful question is not whether the label was canon.
The useful question is:
- what did it actually feel like?
- how did the audio behave?
- what bed / pulse / motion / silence profile produced the result?

We can reconstruct the functional shell even if the label was muddy.

## 6. Design constraints

Do not:
- treat one striking session as decisive
- run too many variables at once
- mix interpretation with scoring during judging
- make the UI theatrically mystical at the cost of usable data

Do:
- preserve raw session notes
- separate capture from judging
- compare against baseline
- use enough trials to make the results worth reading

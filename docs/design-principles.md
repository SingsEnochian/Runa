# Sound Lab Design Principles

A practical blueprint for building Monroe-inspired consciousness audio tools without pretending to duplicate proprietary Monroe audio.

This lab is built on four assumptions:

1. **State architecture matters more than folklore frequency charts.**
   We are designing for repeatable shifts in attention, body-load, imagery, affect, and perceived spaciousness.

2. **A session is a shaped experience, not just a sound bed.**
   Every session should have:
   - descent
   - plateau
   - return

3. **The engine should be modular.**
   We separate:
   - ambient bed
   - entrainment layer
   - movement/modulation layer
   - guidance layer

4. **We test effects, not myths.**
   The lab exists to produce repeatable subjective outcomes with logs and iteration.

---

## 1. Core Design Goal

The goal is not to clone Monroe products.

The goal is to create a system that can reliably generate distinct “state shells” inspired by the Focus-level training model:

- F10 shell: inward, heavy, body-asleep feeling
- F12 shell: wider, brighter, expanded awareness
- F15 shell: thinned sense of time
- F18 shell: warm, affective, heart-centred tone
- F21 shell: threshold / bridge / liminal spaciousness

Each shell should work:
- unguided
- with guidance
- at multiple durations
- with swappable bed textures

---

## 2. Audio Stack

Each session is built from four layers.

### A. Ambient Bed
Purpose:
- create continuity
- reduce environmental harshness
- support immersion

Examples:
- pink-noise wash
- soft filtered broadband air
- low-detail pads
- very light environmental texture

Rules:
- no busy melody during early builds
- avoid sharp transients
- avoid overly narrative sound design
- let the bed support the state, not steal focus

---

### B. Entrainment Layer
Purpose:
- provide repeatable pulse structure
- support state induction

Possible modes:
- binaural beats
- monaural beating
- isochronic pulse
- amplitude-modulated tonal clusters
- hybrid pulse methods

Rules:
- keep it subtle at first
- never make entrainment the whole experience
- treat pulse as one lever among many

---

### C. Movement / Modulation Layer
Purpose:
- create the felt sense of descent, widening, drift, or threshold

Examples:
- slow stereo widening
- slow phase drift
- gentle filter movement
- tiny depth changes
- very low-rate amplitude modulation

Rules:
- motion should be felt more than noticed
- if the listener starts tracking the effect itself, it is too strong

---

### D. Guidance Layer
Purpose:
- direct intention
- cue transitions
- frame exercises

Modes:
- silent
- sparse prompts
- full guided protocol

Rules:
- voice should leave space
- voice should not over-explain
- long pauses are part of the design

---

## 3. Session Envelope

Default structure:

1. **Descent**
   - 5–12 minutes
   - reduce cognitive friction
   - establish bed
   - introduce pulse gradually
   - cue body settling

2. **Plateau**
   - maintain target shell
   - optionally introduce exercise prompt
   - minimal novelty

3. **Return**
   - reduce depth effects
   - slightly brighten the field
   - soften pulse prominence
   - cue re-entry clearly

---

## 4. State Shell Definitions

### F10
Keywords:
- heavy
- warm
- still
- inward
- body-first

Avoid:
- sparkly highs
- dramatic motion
- too much brightness

### F12
Keywords:
- wider
- clearer
- gently alert
- open
- spatial

Avoid:
- sleepiness that collapses into dullness

### F15
Keywords:
- smooth
- timeless
- low event density
- unhurried
- suspended

Avoid:
- obvious markers of progression
- frequent transitions

### F18
Keywords:
- warm
- emotionally coherent
- lifted
- connected
- tender without sentimentality

Avoid:
- manipulative “healing music” clichés
- melodrama

### F21
Keywords:
- threshold
- liminal
- dimensional
- distant
- bridge-state

Avoid:
- instability
- jump-scares
- over-designed eeriness

## 6. Lab Rules

### Rule 1: Start simple
The first successful session should be boring in the best possible way.

### Rule 2: Change one major variable at a time
If you alter:
- bed
- entrainment type
- guidance style
- movement depth

…log the change cleanly.

### Rule 3: Build unguided shells first
A shell should stand on its own before it carries a protocol.

### Rule 4: Keep sessions reproducible
If a session works, we should be able to render it again.

### Rule 5: Log subjective outcomes immediately
Do not trust memory later.

---

## 7. Testing Metrics

After each session, log:

- date/time
- preset used
- headphones used
- session duration
- pre-state
- post-state
- body heaviness (0–10)
- mental clarity (0–10)
- imagery density (0–10)
- emotional warmth (0–10)
- spaciousness (0–10)
- drowsiness / click-out / sleep
- usefulness
- notes

---

## 8. Build Order

Recommended initial order:

1. F10 unguided shell
2. F12 unguided shell
3. F10 guided release protocol
4. F12 guided problem-solving protocol
5. F15 shell
6. F18 shell
7. F21 shell
8. Zener task integration
9. adaptive session engine
10. journaling / stats dashboard

---

## 9. Zener Practice Integration

Zener mode is not used to “prove” anything.
It is used to test whether different sound conditions affect:
- attention
- calm
- image confidence
- response speed
- forced-choice hit rate

The correct attitude is:
- curious
- disciplined
- unsensational

Use:
- randomized targets
- fixed trial counts
- clean hit/miss logging
- baseline comparison across tone conditions

---

## 10. North Star

We are not making occult wallpaper.
We are building a state-shaping audio lab with mythic taste and operational discipline.

If it works, it should work:
- more than once
- under known conditions
- with logs
- with enough clarity to improve it

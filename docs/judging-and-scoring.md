# Judging and Scoring

A practical scoring framework for telepathy-style, remote-viewing-style, and Zener-style experiments in the Sound Lab.

Purpose:
- keep judging cleaner than memory alone
- reduce post-session storytelling drift
- make sessions comparable across time

---

## 1. General Rule

**Capture first. Judge later.**

The person doing the session should not score the session while still emotionally inside it if that can be avoided.

Best case:
- session capture is stored
- target stays hidden until later
- judging happens against decoys

---

## 2. Basic Scoring Types

### A. Hit / Miss
Best for:
- Zener cards
- simple symbol targets
- forced-choice tasks

Use when the target category is discrete and clear.

### B. Rank-Order Judging
Best for:
- image targets
- scene targets
- remote-viewing-style sessions

Method:
- one true target
- 3–5 decoys
- judge ranks best match to worst match

A rank of 1 is strongest.

### C. Descriptor Overlap
Best for:
- sensory targets
- mixed verbal + sketch sessions

Method:
Compare session descriptors against target features.
Examples:
- wet / dry
- open / enclosed
- vertical / flat
- natural / built
- bright / dim

### D. Confidence-Weighted Review
Best for:
- exploratory research
- identifying whether “strong-feeling” hits are more reliable

Method:
Compare:
- average confidence on hits
- average confidence on misses

---

## 3. Recommended Scoring by Protocol

### Zener
Use:
- hit / miss
- total hit rate
- hit rate by confidence
- hit rate by condition

### Ganzfeld-lite
Use:
- blind target judging
- rank-order if target pool contains multiple candidates
- optional descriptor overlap notes

### Remote Viewing
Use:
- rank-order as primary
- descriptor overlap as secondary
- interpretation notes only after primary judging

### F33 Reconstruction Runs
Use:
- state-quality scoring
- task-performance scoring
- compare reconstructed-shell runs against baseline shells

---

## 4. Rank-Order Procedure

Recommended method:

1. Prepare target pool:
   - 1 real target
   - 3 or 4 decoys
2. Present all targets in randomized order
3. Judge the session transcript / sketch against all targets
4. Rank:
   - 1 = strongest match
   - 2 = next best
   - 3 = weaker
   - 4 / 5 = weakest
5. Record rank before revealing the correct answer

Why this is good:
It allows partial information to count without pretending every session is a perfect direct hit.

---

## 5. Descriptor Overlap Grid

For image or scene targets, use a feature grid.

Suggested categories:
- natural / built
- open / enclosed
- bright / dark
- wet / dry
- warm / cool
- active / still
- vertical emphasis / horizontal emphasis
- crowded / sparse
- hard edges / soft edges
- simple / complex

Session output can be compared against target on each category.

Suggested simple scoring:
- 2 = strong match
- 1 = partial match
- 0 = no match

Do not overcomplicate this at first.

---

## 6. Sketch Judging Notes

When sketches are involved, judge:
- major shapes
- relative proportions
- spatial organization
- motion cues
- dominant forms

Do not punish a sketch for not being pretty.
The question is structural resemblance, not artistic merit.

---

## 7. Session Quality Metrics

In addition to target scoring, log state quality:
- body calm 0–10
- clarity 0–10
- imagery density 0–10
- emotional steadiness 0–10
- confidence 0–10
- sense of contact / signal 0–10

Why:
Sometimes a shell improves reporting quality before it improves accuracy.
That is still useful data.

---

## 8. Suggested Session Table

For each session:
- session_id
- protocol_id
- date
- shell
- target_type
- duration
- confidence
- score_type
- raw_score
- normalized_score
- notes

Examples:
- Zener raw_score = 7/25
- Rank-order raw_score = 1 of 4
- Descriptor raw_score = 11/20

---

## 9. Baseline Strategy

Always compare new conditions against something simpler.

Good baseline ladder:
1. silence
2. neutral bed
3. F10 shell
4. F12 shell
5. custom reconstruction shell

This prevents a beautiful custom state from getting credit for effects that were already there.

---

## 10. Error Sources to Watch

Common gremlins:
- target leakage
- judging after feedback
- changing multiple variables at once
- scoring with too much emotional investment
- too few trials
- fatigue masquerading as insight
- retrofitting vague descriptors after seeing the target

If one of these appears, note it plainly.

---

## 11. Minimum Honest Read

A useful honest read after a block is:
- better than chance?
- more confident than baseline?
- more image-rich than baseline?
- easier to judge than baseline?
- worth repeating?

That is enough.

---

## 12. North Star

Good judging should feel a little boring.
That is a compliment.

If the scoring framework is sturdy enough to survive excitement, then it is doing its job.

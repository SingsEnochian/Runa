# Psi Protocols

Practical session formats for telepathy-adjacent and remote-viewing-style experiments inside the Sound Lab.

These protocols are written for:
- calm repeatability
- low ambiguity in setup
- strong separation between capture and judging
- pairing with audio shells without turning the audio itself into the target

---

## 1. General Principles

All psi-style sessions in this repo should follow four rules:

1. **Randomize the target cleanly**
   The target should be selected before the session and hidden from the viewer / receiver.

2. **Capture before interpretation**
   First impressions, sketches, descriptors, and felt qualities come first.
   Interpretation comes later.

3. **Judge blind when possible**
   Wherever possible, compare the session material against:
   - one real target
   - multiple decoys
   without the judge knowing which is correct.

4. **Log conditions every time**
   The audio shell, duration, confidence, and body state matter.

## 2. Recommended Shell Pairing

### F10
Use for:
- calming down before a session
- low-pressure symbol work
- body-heavy receptive states

Best when:
- the user is over-stimulated
- the session goal is quieting noise, not maximizing vividness

### F12
Use for:
- telepathy-style target work
- remote-viewing-style image capture
- Zener and symbol tasks
- open but reportable states

Best when:
- the task needs both calm and alertness
- image-first responses matter

### F15
Use for:
- slow image emergence
- patterning and longer-form drift
- after the user already knows their own response style

Best when:
- the task is exploratory
- timing pressure is low

### F33-ish Reconstruction Shell
Use for:
- reconstructing the feel of a remembered effective state
- testing a personally resonant shell even if the label is unofficial or uncertain

Build target:
- “What did this state actually do for me?”
not
- “Was this label canon?”

Log:
- body feel
- speed of access
- clarity
- imagery
- emotional tone
- sense of distance / threshold / spaciousness

## 3. Protocol: TEL001 Ganzfeld-lite

Purpose:
A stripped-down telepathy-style protocol that can be run at home without elaborate equipment.

### Setup
- one sender (optional, if working in paired mode)
- one receiver
- one hidden target from a pool
- one audio shell, usually F12
- soft visual homogenization if desired
  - closed eyes
  - eye mask
  - dim room
  - low-distraction field

### Flow
1. settle into shell
2. begin target transmission window
3. receiver records:
   - words
   - impressions
   - images
   - body sensations
   - emotions
4. complete session without viewing targets
5. after capture, compare against target pool with blind judging

### Capture Format
- three descriptive words
- one free paragraph
- one optional sketch
- confidence rating 1–5

### Good For
- paired experiments
- testing whether state shells affect perceived target contact
- comparing silence vs tone-shell conditions

## 4. Protocol: RV001 Image-First Remote Viewing

Purpose:
Capture a hidden target without forcing the mind to guess too early.

### Setup
- one hidden target image or location descriptor
- one viewer
- one shell, typically F12
- one fixed session timer

### Flow
1. settle into shell
2. present only target ID or session cue
3. begin image-first phase
4. capture raw material in this order:
   - shapes
   - textures
   - temperature / movement sense
   - colours
   - emotional atmosphere
   - sketch
5. only after raw capture, write one interpretive guess if desired
6. stop before feedback
7. judge later against target + decoys

### Capture Prompts
- hard / soft / open / closed?
- natural / built?
- bright / dim / wet / dry?
- broad / narrow / vertical / flat?
- still / moving / crowded / empty?

### Important Rule
Do not lead with object names.
Lead with sensory structure.

## 5. Protocol: RV002 Rapid Descriptor Session

Purpose:
A shorter-format remote-viewing pass for repeated trials.

### Setup
- hidden target
- short response window
- shell: F12 or neutral bed

### Flow
1. settle briefly
2. show only session cue
3. run 60–120 second descriptor window
4. capture:
   - top 5 descriptors
   - one motion word
   - one colour word
   - one sketch fragment
   - confidence 1–5
5. repeat across multiple targets or sessions

### Good For
- comparing shell conditions
- building trial count faster
- learning personal descriptor habits

## 6. Protocol: ZEN-TEL Hybrid

Purpose:
Blend symbol guessing with image-first capture.

### Flow
1. hidden Zener target selected
2. shell begins, usually F12
3. before choosing symbol, record first impression:
   - shape
   - movement
   - emotional feel
4. choose symbol
5. rate confidence
6. optional note on whether the symbol choice felt direct or translated from an image

### Why This Helps
It distinguishes:
- immediate symbol hit
from
- image-first / translation-first hit

That difference may become important later in analysis.

## 7. Protocol: F33R Reconstruction Session

Purpose:
Rebuild the functional feel of a remembered effective state sometimes labelled “Focus 33.”

This protocol is about reconstruction, not canon enforcement.

### Questions to Answer
- Did the state feel deep or wide?
- Did it make images clearer, quieter, or faster?
- Was it emotionally warm, neutral, or uncanny?
- Was time slowed, blurred, or absent?
- Did it feel closer to F15, F21, or something between?

### Flow
1. run one custom shell
2. log immediate state effects
3. do one short psi task
4. score outcome later
5. iterate only one or two audio variables next round

### Variables to Tune
- spatial depth
- pulse prominence
- bed brightness
- motion rate
- reverb amount
- event density

---

## 8. Minimum Session Log

Every protocol should log:
- date / time
- protocol ID
- shell used
- duration
- headphones / speakers
- pre-state
- post-state
- confidence
- imagery density
- body calm
- target known? no / yes
- judged later? yes / no
- notes

---

## 9. Failure Conditions

Abort or stop early if:
- you become too tired to report clearly
- you feel compelled to force outcomes
- the session becomes more theatrical than useful
- multiple variables changed and the run stops being interpretable

---

## 10. North Star

The best session is not the most dramatic one.
The best session is the one you can learn from and run again.
